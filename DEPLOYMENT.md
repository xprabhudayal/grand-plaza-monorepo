# Hotel Voice AI Concierge - Production Deployment\n\nThis guide explains how to deploy the Hotel Voice AI Concierge application using Docker for production on DigitalOcean or other cloud platforms.\n\n## Prerequisites\n\n- Docker installed on your system\n- Docker Compose (optional but recommended)\n- API keys for the services you want to use\n\n## Building the Docker Image\n\n```bash\ndocker build -t hotel-voice-ai .\n```\n\n## Running with Docker Compose (Recommended)\n\n1. Create a `.env` file with your API keys:\n```bash\ncp .env.example .env\n```\n\n2. Edit the `.env` file and add your API keys:\n```bash\n# Voice Services\nDEEPGRAM_API_KEY=your_deepgram_api_key\nCARTESIA_API_KEY=your_cartesia_api_key\nOPENAI_API_KEY=your_openai_api_key\n# or\nPERPLEXITY_API_KEY=your_perplexity_api_key\n\n# Daily.co Transport\nDAILY_API_KEY=your_daily_api_key\nDAILY_ROOM_URL=your_daily_room_url\n\n# Optional services\nGROQ_API_KEY=your_groq_api_key\nTAVUS_API_KEY=your_tavus_api_key\nTAVUS_REPLICA_ID=your_tavus_replica_id\nTAVUS_PERSONA_ID=your_tavus_persona_id\n```\n\n3. Start the application:\n```bash\ndocker-compose up -d\n```\n\n## Running with Docker Directly\n\n```bash\ndocker run -d \\\n  --name hotel-voice-ai \\\n  -p 80:80 \\\n  -e DEEPGRAM_API_KEY=your_deepgram_api_key \\\n  -e CARTESIA_API_KEY=your_cartesia_api_key \\\n  -e OPENAI_API_KEY=your_openai_api_key \\\n  -e DAILY_API_KEY=your_daily_api_key \\\n  -e DAILY_ROOM_URL=your_daily_room_url \\\n  hotel-voice-ai\n```\n\n## Deploying to DigitalOcean\n\n### Option 1: DigitalOcean App Platform\n\n1. Fork this repository to GitHub\n2. Create a new App in DigitalOcean App Platform\n3. Connect your GitHub repository\n4. Set environment variables in the App Platform dashboard\n5. Deploy!\n\n### Option 2: DigitalOcean Droplet with Docker\n\n1. Create a new Ubuntu Droplet\n2. SSH into the droplet\n3. Install Docker:\n```bash\nsudo apt update\nsudo apt install docker.io docker-compose -y\n```\n\n4. Clone the repository:\n```bash\ngit clone https://github.com/your-repo/hotel-voice-ai-concierge.git\ncd hotel-voice-ai-concierge\n```\n\n5. Create and configure the `.env` file with your API keys\n\n6. Start the application:\n```bash\ndocker-compose up -d\n```\n\n7. Configure firewall to allow HTTP traffic:\n```bash\nsudo ufw allow 80\n```\n\n## Environment Variables\n\n| Variable | Description | Required |\n|----------|-------------|----------|\n| `DEEPGRAM_API_KEY` | Deepgram API key for speech-to-text | Yes (or SONIOX_API_KEY) |\n| `CARTESIA_API_KEY` | Cartesia API key for text-to-speech | Yes |\n| `OPENAI_API_KEY` | OpenAI API key for language model | Yes (or PERPLEXITY_API_KEY) |\n| `DAILY_API_KEY` | Daily.co API key for video transport | Yes |\n| `DAILY_ROOM_URL` | Daily.co room URL for video calls | Yes |\n| `SONIOX_API_KEY` | Alternative STT service | Optional |\n| `PERPLEXITY_API_KEY` | Alternative LLM service | Optional |\n| `GROQ_API_KEY` | Alternative LLM service | Optional |\n\n## Persistence\n\nThe application uses SQLite for data storage. In the Docker Compose setup, the database file is mounted to a named volume for persistence. If running with Docker directly, you may want to mount a volume for the database file:\n\n```bash\ndocker run -d \\\n  --name hotel-voice-ai \\\n  -p 80:80 \\\n  -v /path/to/database.db:/home/app/backend/database.db \\\n  -e DEEPGRAM_API_KEY=your_key \\\n  # ... other environment variables\n  hotel-voice-ai\n```\n\n## Accessing the Application\n\nOnce deployed, access the application at:\n- Frontend: `http://your-server-ip` or your domain\n- API Docs: `http://your-server-ip/docs`\n- Health Check: `http://your-server-ip/health`\n\n## Updating the Application\n\nTo update the application:\n\n1. Pull the latest code:\n```bash\ngit pull origin main\n```\n\n2. Rebuild and restart:\n```bash\ndocker-compose down\ndocker-compose up --build -d\n```
